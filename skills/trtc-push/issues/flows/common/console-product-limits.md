# 产品能力 / 套餐 / 控制台限制咨询处理流程

## 入口现象

用户咨询套餐版本、证书数量、控制台入口、推送总量、QPS、角标、撤回、定时、全员推送统计
等产品能力边界，或误把能力限制当成接入故障。

这类问题优先按 FAQ / 产品边界处理，不要直接进入 SDK 接入排障。

## 首轮证据

- 用户目标能力是什么。
- 当前 `SDKAppID` 和套餐版本。
- 控制台入口、报错截图或错误码。
- 平台 / 厂商 / 证书数量。
- 是否涉及厂商侧权益、消息分类、额度或审核。
- 是否已有运行时日志证明是接入故障。

## 排查顺序

1. 先判断是产品咨询、控制台操作问题，还是真实推送故障。
2. 如果没有运行时异常，只按产品能力和控制台规则回答。
3. 确认套餐版本和目标能力是否匹配。
4. 对证书数量、厂商额度、QPS、控制台入口等给出明确边界。
5. 对不支持能力给出替代方案或业务规避方案。
6. 如果用户同时反馈真实收不到，再转对应平台排查流程。

## 分支处理

| 分支 | 判断信号 | 处理动作 |
|---|---|---|
| 证书数量上限 | `cert limit exceeded`、证书数量已达上限 | 查 `../../cards/common/console-certificate-quota.md` |
| 推送总量 / QPS | 咨询总量限制、解除限制 | 区分腾讯云 QPS 与厂商每日额度 |
| 控制台入口 / 查询方式 | 找不到入口或查不到记录 | 指引正确控制台路径和查询条件 |
| 产品不支持 | 角标图片、部分厂商角标、特殊控制能力 | 明确不支持并给替代方案 |
| 厂商权益 | 即时通信分类、私信通道、厂商限流 | 引导用户到厂商控制台申请或确认 |

## 验证信号

- 用户确认问题属于能力边界、套餐限制或控制台操作。
- 已给出可执行的升级、配置、申请或替代方案。
- 若转入故障排查，已进入对应 flow 并收集必要证据。

## 何时升级 / 转交

- 需要升级套餐、调整计费或开通额外能力。
- 控制台当前不支持用户所需配置。
- 需要产品确认未来排期或能力边界。
