# 角标 / Badge 显示、清除和厂商差异排查流程

## 入口现象

用户反馈通知角标不显示、角标不清零、角标数字不符合预期、希望设置角标图片，或不同厂商
角标表现不一致。

角标问题高频但不是同一根因。它可能是产品能力边界、厂商系统限制、payload 参数、客户端
清零逻辑或 iOS / Android 差异。

## 首轮证据

- 平台和厂商：iOS / Huawei / Honor / OPPO / vivo / Xiaomi / Samsung 等。
- 期望能力：显示数字、红点、图片、自定义、点击清零、打开 App 清零。
- 推送 payload：`BadgeMode`、`BadgeAddNum`、APNs badge、厂商 badge 字段。
- 通知是否实际展示。
- 客户端是否实现清零逻辑。
- 系统桌面是否支持对应角标能力。

## 排查顺序

1. 先确认目标平台 / 厂商是否支持该角标能力。
2. 确认用户期望的是数字、红点、图片还是清零。
3. 检查服务端 payload 是否携带正确 badge 参数。
4. iOS 检查 APNs badge 字段和客户端清零逻辑。
5. Android 按厂商确认数字角标、自定义角标和清除策略。
6. 如果厂商不支持，按能力边界答复，不继续要求用户改配置。

## 分支处理

| 分支 | 判断信号 | 处理动作 |
|---|---|---|
| 产品不支持 | 角标图片、部分厂商数字角标 | 明确能力边界，给替代方案 |
| payload 配置错误 | `BadgeMode` / `BadgeAddNum` / APNs badge 缺失 | 修正服务端参数 |
| 客户端未清零 | 点击或打开 App 后角标仍在 | 客户端调用系统或厂商 API 清零 |
| 厂商系统行为 | 三星跟随通知栏、OPPO/vivo 限制 | 按厂商规则解释 |
| 通知未展示 | 角标依赖通知展示 | 先排查通知是否展示 |

## 验证信号

- 支持角标的厂商上，payload 修正后角标按预期变化。
- 客户端清零逻辑执行后，桌面角标消失或归零。
- 不支持场景能给出明确边界和替代方案。

## 何时升级 / 转交

- 厂商公开文档未说明角标行为。
- 同一系统版本不同设备表现不一致。
- 需要厂商确认桌面角标能力或审核权益。
