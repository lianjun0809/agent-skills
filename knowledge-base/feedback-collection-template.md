# TRTC Slice 信息收集表格

> 本表格用于产品/销售团队收集用户高频问题，转化为知识库 slice。
> 请按照下方列定义填写，每行一个功能点/问题。

---

## 列定义

| # | 列名 | 填写说明 | 必填 | 转化用途 |
|---|------|---------|------|---------|
| 1 | **功能点名称** | 用一句短语描述功能，如"消息发送失败" | ✅ | → slice 名称 |
| 2 | **所属产品** | 下拉选择：Chat / Call / RTC Engine / Live / Room | ✅ | → 产品分类 |
| 3 | **涉及平台** | 多选：Web / Android / iOS / Flutter / 全平台 | ✅ | → 平台覆盖 |
| 4 | **类型** | 下拉选择：🔥高频使用 / ❌高频出错 / 🔥❌两者都是 | ✅ | → slice 侧重方向 |
| 5 | **频次/严重度** | 下拉选择：高 / 中 / 低 | ✅ | → 优先级排序 |
| 6 | **用户典型问题描述** | 用户原话或工单摘要，1-3 句话 | ✅ | → 排障「问题现象」 |
| 7 | **根因/解决方案** | 如果知道就填，不知道留空 | 选填 | → 排障流程 |
| 8 | **相关 API / 错误码** | 关键 API 名或错误码，如 `sendMessage`、`6206` | 选填 | → 代码示例/错误码表 |
| 9 | **现有文档链接** | 官方文档 URL | 选填 | → 参考文档 |
| 10 | **填写人 / 来源** | 谁填的，来自哪个渠道（工单/群聊/客户反馈等） | ✅ | → 后续追溯 |

## 处理工作流

```
产品/销售填表
  → 匹配已有 slice：这个问题属于哪个主线 slice？
     ├─ 能归入 → 作为该 slice 的「真实案例」补充进排障指南
     └─ 不能归入 → 创建新的反馈 slice（source: feedback）
  → 排优先级：(两者都是 > 出错 > 使用) × (高 > 中 > 低)
  → 按优先级编写 slice
```

### 优先级矩阵

|  | 🔥❌ 两者都是 | ❌ 高频出错 | 🔥 高频使用 |
|--|-------------|-----------|-----------|
| **高** | P0 — 立即处理 | P0 — 立即处理 | P1 — 优先处理 |
| **中** | P1 — 优先处理 | P1 — 优先处理 | P2 — 排期处理 |
| **低** | P1 — 优先处理 | P2 — 排期处理 | P2 — 排期处理 |
